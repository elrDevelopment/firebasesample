Caching Data & Transactions

Caching provides the user with a more reactive experience by provide data instantly versus waiting. We want to reduce wait time were we can. Only using loading or spinners when we really have to.

  • User Profile data Lazy Cache

    Caching the request response using simple IF statement.

  • Using LocalStorage API

    Local storage is a key/value index that provides persistence - but is wiped when the user clears site data.

  • IndexedDb

    Key - Document store. No SQL implementation on the client browswer. And is supported on all major browsers.
